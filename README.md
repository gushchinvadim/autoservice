АвтоСервис — это полнофункциональное веб-приложение для учёта технического обслуживания автомобилей.
Система позволяет вести историю ТО, прогнозировать следующие обслуживания, рассчитывать стоимость запчастей и работ,
а также визуализировать историю замен в виде плоттера.

## 🔮 Прогноз следующего ТО

- Автоматический расчёт следующего ТО с учётом цикличности (например, цикл 240 000 км с шагом 15 000 км)

- Разделение стоимости на запчасти и работы
- Список покупок с ценами и количеством
- Учёт специфичных и универсальных нормативов для разных моделей авто

## 📊 Плоттер замен (Матрица истории)

- Зелёные ячейки с датами замен
- Прилипающие заголовки при прокрутке
- Быстрый поиск «когда последний раз менялся аккумулятор»

## 🚀 Установка и запуск

1. Клонирование репозитория
git clone https://github.com/ВАШ_ЛОГИН/autoservice.git
cd autoservice
2. Backend (Django)

#### Создание виртуального окружения
python -m venv .venv
source .venv/bin/activate  # macOS/Linux
#### .venv\Scripts\activate   # Windows

#### Установка зависимостей
pip install django djangorestframework djangorestframework-simplejwt \
            django-cors-headers django-filter pandas openpyxl

#### Применение миграций
python manage.py migrate

#### Создание суперпользователя
python manage.py createsuperuser

3. Frontend (React)

cd autoservice_frontend
npm install

4. Запуск
### Терминал 1 — Backend:
python manage.py runserver
#### Сервер запустится на http://127.0.0.1:8000/

### Терминал 2 — Frontend:
cd autoservice_frontend
npm run dev
#### Приложение откроется на http://localhost:5173/

## ⚙️ Первичная настройка

1. Создание цикла ТО
    Без цикла ТО система не сможет рассчитывать прогноз.
        1. Откройте админку: http://127.0.0.1:8000/admin/
        2. Перейдите в Циклы ТО → Добавить цикл ТО
        3. Заполните:
            - Название: Универсальный
            - Для модели авто: оставьте пустым (универсальный для всех авто)
            - Длина цикла (км): 240000
            - Шаг между ТО (км): 15000
            - Активен: ✅
        4. Сохраните
2. Настройка CORS (уже выполнена)
В autoservice_backend/settings.py разрешён доступ с http://localhost:5173.
   3. Настройка JWT (уже выполнена)
   - Access токен живёт 30 минут
   - Refresh токен — 7 дней
   - Автоматическое обновление при истечении access
## 📖 Использование системы
### Добавление автомобиля
    - Войдите в систему (логин/пароль суперпользователя)
    - Нажмите 🚗 Автомобили в меню
    - Нажмите + Добавить авто
    - Заполните модель, VIN, пробег, дату регистрации
    - Сохраните

### Создание записи о ТО
    - Откройте страницу автомобиля
    - Нажмите + Добавить запись о ТО
    - Укажите дату и пробег
    - Отметьте выполненные работы (чекбоксы)
    - Добавьте использованные запчасти — цена подставится автоматически
    - При необходимости измените количество и цену
    - Итоговая сумма пересчитается мгновенно
    - Нажмите Сохранить запись о ТО

### Просмотр прогноза
    На странице автомобиля автоматически отображается:
        - Следующее ТО (например, ТО-180)
        - Пробег, на котором оно состоится
        - Оставшиеся километры
        - Список запланированных работ с ценами
        - Список запчастей к покупке
        - Итоговая стоимость (запчасти + работы)

### Плоттер замен
        Внизу страницы автомобиля — матрица истории:
        - Строки: все когда-либо заменённые запчасти и работы
        - Столбцы: проведённые ТО (ТО-75, ТО-90, ТО-120...)
        - Зелёные ячейки: дата замены
        - Прочерки: элемент не менялся

### Импорт данных из Excel
        Формат файла запчастей (autoservice_parts.xlsx)
        Формат файла регламентов (autoservice_maintenance.xlsx)

### Как импортировать

        Админка → Запчасти → кнопка 📥 Импорт запчастей из Excel
        Админка → Регламентные операции → кнопка 📥 Импорт регламентов из Excel

### 🔌 API Endpoints

    /api/token/ Получить JWT токены
    /api/token/refresh/ Обновить access токен
    /api/cars/ Список/создание авто
    /api/cars/{id}/ Детали авто
    /api/cars/{id}/forecast/ Прогноз следующего ТО
    /api/tasks/ Регламентные операции
    /api/records/ Записи о ТО (фильтр ?car={id})
    /api/spareparts/ Справочник запчастей
    /api/car-models/ Модели автомобилей

## 🔐 Аутентификация
    
    Система использует JWT (JSON Web Tokens):
    Пользователь вводит логин/пароль на странице /login
    Django возвращает access (30 мин) и refresh (7 дней) токены
    Токены сохраняются в localStorage
    Axios автоматически подставляет Authorization: Bearer <token> в каждый запрос
    При истечении access токена — автоматический refresh
    При истечении refresh — редирект на страницу входа
    Учётные данные по умолчанию: создаются командой python manage.py createsuperuser
